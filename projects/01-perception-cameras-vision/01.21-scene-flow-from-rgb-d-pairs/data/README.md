# Data — 01.21 Scene flow from RGB-D pairs

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

### What is rendered

A camera at `CAM_POS0 = (0, 0, 1.4)` m (world/body frame: x-forward, y-left, z-up, no tilt) looks
down a small scene: a hashed-texture ground plane (z=0), a hashed-texture back wall 9 m ahead
(tall enough to fill the whole vertical field of view — no wall/sky seam is visible, see
`make_synthetic.py`'s `WALL_Z_TOP` comment for why that matters to Lucas-Kanade), two static
textured boxes, and **one independently moving textured box**. Every surface uses the same 3-D
value-noise texture function (`value_noise3`) so dense optical flow has gradient almost everywhere.

Two exposures are rendered: t0 (`frame0_*`) and t1 (`frame1_*`), related by a **known camera
ego-motion** — see the table below — with the moving box additionally translating by a **known
independent motion**. A third exposure, `static_frame1_*`, renders the SAME camera ego-motion with
the box left in its t0 position — the negative-control pair `main.cu`'s `static_negative_control`
gate uses to check the pipeline does not hallucinate movers when none exist.

| Quantity | Value | Meaning |
|----------|-------|---------|
| Ego rotation | 3.000° yaw (world +z) | camera1's orientation relative to camera0, world/body frame |
| Ego translation | 0.090 m, world +x (forward) | camera1's position relative to camera0 |
| Object translation | 0.300 m, world +y (left) | the moving box's world-frame displacement, t0→t1 |
| Depth sensor noise | `sigma_z(z) = 0.0015 + 0.00015*z^2` m | disparity-quantization model, THEORY.md derives it |

`kernels.cuh`'s `kTGt`/`kCGt` constants (in `main.cu`, actually — the ground-truth *camera-optical-
frame* rigid transform and object-residual offset the pipeline's gates compare against) are
**manually mirrored** from this script's own printed derivation (run it and read the last few
lines) — see `make_synthetic.py`'s module docstring for the full frame-conjugation derivation
(this is *not* simply "R_ego, transposed" — the docstring explains why, including the bug this
project's own build caught when that conjugation was first missed).

### Fields / format

| File | Format | Shape | Units / frame |
|------|--------|-------|----------------|
| `frame0_rgb.ppm` | binary P6 PPM | 128×96×3 uint8 | camera0 image, sRGB-ish (no gamma correction applied — the renderer's shading is already in display-ready 0–255 range) |
| `frame1_rgb.ppm` | binary P6 PPM | 128×96×3 uint8 | camera1 image (dynamic pair: camera AND object moved) |
| `static_frame1_rgb.ppm` | binary P6 PPM | 128×96×3 uint8 | camera1 image (negative-control pair: camera moved, object did NOT) |
| `frame0_depth.bin` | raw float32, row-major | 128×96 | meters, camera0 OPTICAL frame (Pcam.z, pinhole convention); `-1.0` = sky (no return) |
| `frame1_depth.bin` | raw float32, row-major | 128×96 | meters, camera1 OPTICAL frame, dynamic pair |
| `static_frame1_depth.bin` | raw float32, row-major | 128×96 | meters, camera1 OPTICAL frame, negative-control pair |
| `truth_flow.bin` | raw float32, row-major, interleaved (u,v) | 128×96×2 | pixels, dense 2-D optical flow frame0→frame1 (0,0 where no truth exists, i.e. sky in frame0) |
| `truth_scene_flow.bin` | raw float32, row-major, interleaved (x,y,z) | 128×96×3 | meters, the exact "raw" 3-D scene flow `P2_cam1 - P1_cam0` (mixed-frame, see `kernels.cuh`'s file header for what this means and why) |
| `truth_mask.pgm` | binary P5 PGM | 128×96 uint8 | 255 = this frame0 pixel's ray hit the moving object, 0 = everything else (background or sky) |

`frame0_*` is shared by BOTH the dynamic and negative-control pairs (the object has not moved yet
at t0 either way) — there is no separate `static_frame0_*` file (would be a byte-for-byte
duplicate of `frame0_*`, against the "tiny sample" spirit of CLAUDE.md §8).

### Measured coverage (from an actual `make_synthetic.py` run, seed 42)

- 12288/12288 pixels (100%) hit the scene in frame0 (the tall wall removes any wall/sky seam —
  see above); 301 pixels (2.4%) are the moving object.
- 12288/12288 frame0 pixels have a valid dense truth correspondence into camera1 (none ran off the
  back of camera1 or behind it, at this motion scale).

# Data — 01.11 Low-light denoising (bilateral, non-local means, fast BM3D variant)

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

### Files

| File | Format | What it is |
|---|---|---|
| `clean.pgm` | Binary PGM (P5), 200x150, 8-bit grayscale | The NOISE-FREE ground truth: a 3-octave hashed multi-scale texture background, three flat (constant-radiance) patches at dark/mid/bright code values, one high-contrast step edge, and one fine-detail stripe ruling (`src/kernels.cuh` Section 3 names every rectangle). Never seen by any denoiser — read ONLY by `main.cu`'s five gates. |
| `noisy.pgm` | Binary PGM (P5), 200x150, 8-bit grayscale | The ONLY frame every denoiser (bilateral/NLM/BM3D-lite/Gaussian baseline) actually sees: `clean.pgm` through this project's low-light sensor model — exact Poisson shot noise (Knuth's inversion algorithm, xorshift32-driven) at each pixel's expected electron count (`clean_dn / kDnPerElectron`, peak 40 e- at code value 255) plus additive Gaussian read noise (sigma=2.0 e- rms), converted back to DN and quantized to 8 bits. |
| `params.csv` | CSV, `key,value` rows | Every generation constant: image size, the noise model's `PEAK_ELECTRONS`/`READ_NOISE_E`/`DN_PER_ELECTRON`, the three texture octaves (cell size/amplitude/hash salt), and every scene rectangle (cross-referenced in `src/kernels.cuh` by the same names). |

### The model, in brief (full derivation in `THEORY.md` and the generator's own comments)

`noisy_dn = clamp_round(( Poisson(clean_dn / kDnPerElectron) + N(0, kReadNoiseE^2) ) * kDnPerElectron)`,
where `kDnPerElectron = 255/kPeakElectrons = 6.375 DN/e-` (`kPeakElectrons = 40` electrons at code
value 255 — a deliberately extreme low-light operating point, task brief) and `kReadNoiseE = 2.0`
electrons rms. Poisson sampling is EXACT (Knuth's multiplicative-inversion algorithm, not the Gaussian
shot-noise approximation — `scripts/make_synthetic.py`'s module docstring justifies the choice: at this
operating point the darkest committed flat patch has an expected signal of only ~4.4 electrons, exactly
where a Gaussian approximation would visibly disagree with the true discrete, right-skewed, non-negative
Poisson distribution). Measured noisy-frame std in the three flat patches (dark/mid/bright, code values
28/128/175) tracks the analytic prediction `sqrt(signal_e + kReadNoiseE^2) * kDnPerElectron` within a few
percent (`main.cu`'s `noise_model_sanity` gate; measured ratios 1.03/0.92/0.89 — see README "Expected
output").

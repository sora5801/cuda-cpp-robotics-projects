# Data — 01.08 HDR exposure fusion + tone mapping for outdoor robots

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

### Files

| File | Format | What it is |
|---|---|---|
| `exposure_0.pgm` .. `exposure_3.pgm` | Binary PGM (P5), 160x120, 8-bit grayscale | The four bracketed LDR exposures, shortest (`exposure_0`, t=1/1000 s) to longest (`exposure_3`, t=1/8 s) — see `params.csv` for the exact `T{i}_seconds` value of each. Clipped (0/255) and noisy, exactly as a real sensor would produce. |
| `ground_truth_radiance.bin` | Raw binary, **no header** | 160*120 IEEE-754 `float32` values, row-major (`y*160+x`), little-endian. The EXACT, noise-free, pixel-integrated scene radiance (relative synthetic units — see `THEORY.md` "The problem" for why these are not calibrated photometric units) that produced every exposure above. The oracle for `main.cu`'s `radiance_reconstruction` gate. |
| `params.csv` | CSV, `key,value` rows | Every generation parameter: image size, exposure times, the known CRF's `CRF_GAMMA`/`CRF_S_HALF`, every scene radiance tier, the noise model's sigma, the RNG seed, and the gate ROI/scanline coordinates `main.cu` also uses (cross-referenced there by the same names). |

### The scene, in brief (full derivation in `THEORY.md` and the generator's own comments)

A single-channel (grayscale — see README "Limitations & honesty" for why) 160x120 outdoor scene: a sun
disk and sky band (top), an open-shade band, and a sunlit-concrete band containing a painted line
marking and a deep "under-vehicle" shadow rectangle, all textured with smooth hashed-lattice noise so
detail exists everywhere (including inside the shadow) — plus one noise-free, purely analytic monotonic
radiance ramp (rows 110-117) used only by the `tone_map_range` gate's monotonicity check. Radiance spans
`R_SHADOW=2.0` to `R_SUN=200000.0` (relative units, a ratio of `1e5`, ~5 decades). Rendered through a
Naka-Rushton/Michaelis-Menten camera response function (`CRF_GAMMA=0.85`, `CRF_S_HALF=3.0`) with mild
additive sensor noise (`NOISE_SIGMA_CODE=1.5` code-value units), then rounded and clipped to 8 bits.

# Data — 01.09 Photometric/vignetting calibration kernels

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

### Files

| File | Format | What it is |
|---|---|---|
| `dark_00.pgm` .. `dark_15.pgm` | Binary PGM (P5), 160x120, 8-bit grayscale | The 16-frame DARK stack: aperture closed, `L=0` everywhere. Expected pixel value is the additive field `o(x,y)` alone (black level + hashed DSNU), plus per-frame read noise. Averaging these 16 recovers `o(x,y)` — the `stack_mean` calibration stage's input. |
| `flat_00.pgm` .. `flat_15.pgm` | Binary PGM (P5), 160x120, 8-bit grayscale | The 16-frame FLAT stack: uniform illumination `L=L_FLAT=180.0`. Expected pixel value is `g(x,y)*L_FLAT + o(x,y)` plus per-frame shot+read noise. Averaging, dark-subtracting, and center-normalizing these 16 recovers the nonparametric gain map. |
| `scene.pgm` | Binary PGM (P5), 160x120, 8-bit grayscale | ONE natural test scene: a textured background plus five identical-radiance (`L_SWATCH=200.0`) flat "gray card" swatches (one centered, four near the corners) rendered through the SAME sensor model. The `correction_efficacy` gate's input. |
| `gain_true.bin` | Raw binary, **no header** | 160*120 IEEE-754 `float32`, row-major (`y*160+x`), little-endian. The EXACT multiplicative field `g(x,y) = V(x,y)*PRNU(x,y)` — the `gain_recovery` gate's oracle. |
| `dsnu_true.bin` | Raw binary, **no header** | 160*120 IEEE-754 `float32`, row-major, little-endian. The EXACT additive field `o(x,y) = BLACK_LEVEL + DSNU_hashed(x,y)` — the `dsnu_recovery` gate's oracle. |
| `params.csv` | CSV, `key,value` rows | Every generation parameter: image size, stack sizes, the vignette's `FOCAL_EFF_PX`/`CENTER_OFFSET_*_PX`, the PRNU/DSNU amplitudes, the noise model's sigmas, `L_FLAT`, the five `SWATCH_*` rectangles (cross-referenced in `src/main.cu` by the same names), and the RNG seed. |

### The model, in brief (full derivation in `THEORY.md` and the generator's own comments)

`I(x,y) = g(x,y)*L(x,y) + o(x,y) + noise`, where `g = V*PRNU` (a cos^4-law optical vignette, focal ratio
`FOCAL_EFF_PX=200.0` with a small deliberate decentering, times a smooth-plus-hashed PRNU field,
amplitude ~+-2%) and `o = DSNU` (`BLACK_LEVEL=8.0` plus a hashed per-pixel offset, amplitude ~+-2 LSB).
Noise is signal-dependent Gaussian (read floor `READ_NOISE_SIGMA=1.2` + shot term
`SHOT_NOISE_K=0.05*sqrt(light_signal)`), drawn independently per frame with a fixed per-frame seed
derived from `SEED=42`. The measured recovered-gain range on the reference machine is `[0.627, 1.014]` —
a ~37% worst-corner falloff, consistent with the vignette-heavy corners a real wide-angle lens produces.

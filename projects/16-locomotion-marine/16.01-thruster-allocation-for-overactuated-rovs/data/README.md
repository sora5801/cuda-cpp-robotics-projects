# Data — 16.01 Thruster allocation for overactuated ROVs (batched QP)

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

### `wrench_batch.csv` — the file the program actually loads at runtime

A synthetic **50-second, 10 Hz commanded-wrench trajectory** (500 rows): an ROV holding station against
a fluctuating current (small always-on sinusoidal + seeded-random disturbance on every channel), plus
two deliberate raised-cosine "docking correction burst" pulses (peaking at `t=15s` and `t=33s`) sized to
saturate several thrusters — see `../src/scripts/make_synthetic.py`'s `make_wrench_row` docstring for the
exact formula and THEORY.md for why the batch needs both unsaturated and saturated rows. Loaded by
`../src/main.cu`'s `load_wrench_batch`.

| Column | Type | Units | Frame | Meaning |
|---|---|---|---|---|
| `t_s` | float | s | — | sample timestamp, monotonic, `dt=0.1s` |
| `Fx_N` | float | N | body | commanded surge force |
| `Fy_N` | float | N | body | commanded sway force |
| `Fz_N` | float | N | body | commanded heave force |
| `Mx_Nm` | float | N·m | body | commanded roll moment |
| `My_Nm` | float | N·m | body | commanded pitch moment |
| `Mz_Nm` | float | N·m | body | commanded yaw moment |

Body frame: **x-forward, y-starboard, z-down** (Fossen/SNAME marine convention — a deliberate,
documented deviation from the repo's default `x-forward/y-left/z-up`; see `../src/kernels.cuh`'s header
comment for the full reasoning). Of the 500 rows, 472 come out unsaturated and 28 saturate at least one
thruster under this project's QP (measured by `main.cu`, printed as the `[info] batch composition` line).

### `rov_geometry.csv` — human-readable documentation, NOT read at runtime

A plain-text mirror of the 8 thruster positions/directions/limits that are actually compiled into
`../src/kernels.cuh` as `constexpr` tables — **the program's real source of truth for the vehicle's
geometry** (CLAUDE.md §12: state is single-sourced). This CSV exists purely so a learner can inspect the
vehicle's geometry without reading C++; it is regenerated by the same script that mirrors the numbers,
but there is no automated check that the two stay in sync — an honest, documented limitation (README
"Limitations": if you change one, update the other by hand).

| Column | Type | Units | Frame | Meaning |
|---|---|---|---|---|
| `thruster` | string | — | — | thruster id (`H1..H4` horizontal, `V1..V4` vertical) |
| `x_m,y_m,z_m` | float | m | body | thruster mount position `r_i` |
| `dx,dy,dz` | float | unitless | body | thruster unit thrust direction `d_i` |
| `u_max_N` | float | N | — | per-thruster saturation limit (symmetric ±) |

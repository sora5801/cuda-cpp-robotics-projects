# Why the DETECT:/LOCALIZE:/RIPENESS: lines below carry NO measured numbers, only thresholds and
# verdicts: two of the pipeline's per-component statistics (mean hue -> ripeness, mean depth ->
# localization) are built from GPU atomicAdd on FLOATS, and float addition is not associative — in
# principle the exact bit pattern of a sum can depend on thread-scheduling order. The measured
# values have been bit-stable across every run and both Debug/Release configurations on the
# reference GPU, but this project follows 08.01's precedent: promise the VERDICT against a fixed,
# documented threshold, print the actually-measured number on the unchecked "[info]" line right
# above it. See src/main.cu's "NOTE on determinism" comment.

# Data — 02.04 Euclidean clustering via GPU union-find / connected components

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

## What the scene is

A ground layer (180,000 points, a flat 60 m × 30 m plane with millimeter-scale jitter — playing the role
of "everything project 02.03, ground segmentation, would already have removed") plus 1,469 **non-ground**
points, the actual clustering input, assembled into geometric scenarios chosen to make four specific
claims about Euclidean clustering checkable (see `../scripts/make_synthetic.py`'s module docstring for
the full scene map and `../THEORY.md` for why each scenario matters):

- **Five generic obstacles** (boxes and poles, filled synthetic point blocks — see the honest scoping
  note below) at generous mutual separation, in a dedicated Y-band.
- **A separation-test pair**: two poles kept exactly `d + 0.10 m` apart, edge to edge — the ground truth
  puts them in two DIFFERENT connected components, and the pipeline must agree.
- **A chaining-test pair**: two boxes bridged by a thin single-file line of points spaced `d - 0.10 m`
  apart — the ground truth MERGES them into one connected component (the honest single-linkage
  "chaining" failure mode, `../THEORY.md` "The problem"), and the pipeline must reproduce that merge,
  not "fix" it.
- **The long snake**: a gently curved arc of 299 points spaced just under `d` apart (a near-exact path
  graph, diameter ≈ 298 hops) — engineered specifically to make label propagation's `O(diameter)` sweep
  count and union-find's `O(log diameter)` sweep count visibly, measurably different on the same data.
- **28 scattered noise points**, each placed on a coarse 2 m grid (far more than `2*d` from every other
  point, guaranteeing each is its own singleton connected component) — must be rejected by min-size
  filtering.

**Measured composition of the committed file** (an actual `make_synthetic.py` run, not estimated): 37
raw single-linkage connected components → 9 would be reported after `min_cluster_size = 5` filtering
(28 points rejected as noise). Ground-truth cluster ids are computed by the SAME distance rule and the
SAME 27-cell-voxel-stencil algorithm the C++/CUDA pipeline implements, run independently in pure Python
(`build_truth_clusters()`) — see `../THEORY.md` "How we verify correctness" for why this specific
choice (truth computed the same way, by a genuinely separate program, before the pipeline exists) is a
particularly strong correctness anchor.

**Honest scoping note:** the non-ground objects are FILLED synthetic point lattices (a box or cylinder
sampled on a jittered 3-D grid), not ray-cast LiDAR returns — this project's teaching focus is the
CLUSTERING algorithm, not sensor simulation (that realism is 02.01's / 11.01's / 01.18's job, cited in
`../README.md`); a real scan's point density and surface-only (not filled-volume) sampling pattern would
differ. The ground layer's own extent does not need to bound the snake's arc — the demo's topview
artifacts derive their world-to-pixel window from the union of every rendered point, not the ground
layer alone (`../src/main.cu`), so nothing is silently clipped.

### Fields / format (`cluster_scene.bin`)

A small hand-rolled binary format (no library — CLAUDE.md's no-black-boxes stance), read by
`../src/main.cu`'s `load_scene()` with **explicit fixed-width primitive reads** (never a raw struct
`fread`, so the layout never depends on a compiler's struct-padding rules):

| Bytes | Type | Field | Meaning |
|-------|------|-------|---------|
| 0–7 | `char[8]` | magic | literal `CLUSTR01`, no null terminator |
| 8–11 | `int32` | `n_ground` | ground-layer point count (180,000) — context only, never clustered |
| 12–15 | `int32` | `n_nonground` | the clustering input's point count (1,469) |
| 16–19 | `float32` | `d_m` | cluster tolerance = voxel leaf (0.40 m) — **must equal** `kernels.cuh`'s `kClusterToleranceM`; `main.cu` asserts this at load time |
| 20–23 | `int32` | `min_cluster_size` | noise-rejection threshold (5) — must equal `kernels.cuh`'s `kMinClusterSize` |
| 24–27 | `int32` | `snake_start_idx` | index (into the non-ground array) where the long-snake chain begins |
| 28–31 | `int32` | `snake_count` | number of snake points (299) |
| 32–35 | `int32` | `sep_a_idx` | a representative point index inside the separation-test's first object |
| 36–39 | `int32` | `sep_b_idx` | a representative point index inside the separation-test's second object |
| 40–43 | `int32` | `chain_a_idx` | a representative point index inside the chaining-test's first object |
| 44–47 | `int32` | `chain_b_idx` | a representative point index inside the chaining-test's second object |
| 48–51 | `int32` | `noise_start_idx` | index where the scattered noise points begin |
| 52–55 | `int32` | `noise_count` | number of noise points (28) |
| 56–59 | `int32` | reserved | always 0 |
| 60 … | `float32[n_ground*3]` | ground `xyz` | interleaved x,y,z, **meters**, local/"sensor" frame (+x forward, +y left, +z up — CLAUDE.md §12), context/visualization only |
| … | `float32[n_nonground*3]` | non-ground `xyz` | interleaved x,y,z, meters, same frame — **the clustering input** |
| … | `int32[n_nonground]` | `truth_cluster_id` | ground-truth canonical cluster id per non-ground point (the MINIMUM non-ground-array index in that point's true connected component; `-1` never appears here — every point belongs to some component, even a noise singleton) |

> **Synthetic, labeled everywhere.** Every point in this file is analytically generated (filled lattice
> sampling, arc sampling, or a coarse isolated grid) — nothing here is a recording of any real sensor or
> real space. `main.cu`'s `DATA:` line and every artifact this project writes say so explicitly.

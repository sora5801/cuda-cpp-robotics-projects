# Data — 02.09 Normal + curvature estimation at millions of points/sec

Provenance, licensing, and field documentation for everything under `data/` (CLAUDE.md §8).

## The rules (repo-wide, CLAUDE.md §8)

- **Synthetic-first.** Robotics data can almost always be synthesized with full ground truth;
  `../scripts/make_synthetic.py` is the default source, and synthetic data is **labeled synthetic
  everywhere it appears**.
- **Tiny committed sample.** `sample/` holds just enough committed data that the demo runs offline
  with zero downloads. Large/raw/downloaded data lives in `downloaded/` (git-ignored).
- **Public datasets** (where one genuinely teaches more) are fetched by `../scripts/download_data.ps1`
  / `.sh` — idempotent, with source URL, expected size, and checksum documented below. **Respect every
  license**; registration-gated or no-redistribution datasets (KITTI, nuScenes) are pointed at, never
  mirrored.
- **Never fabricate.** No made-up measurements, no invented ground truth passed off as real.

## This project's data

Normal/curvature estimation can only be graded honestly against a **truth engine** — a surface whose
normal and curvature are known in closed form at every point. No public LiDAR dataset ships that ground
truth, so this project is synthetic-only; no `download_data.*` script applies (`scripts/download_data.ps1`
/ `.sh` are stubs that say so explicitly).

## What the scene is

A single simulated sensor sits at the origin (0, 0, 0), +x forward (CLAUDE.md §12's LiDAR-frame
convention). Four **analytic surfaces** — a tilted **plane**, a **sphere**, a **cylinder**, and a sharp
two-plane **edge** (a right-angle corner) — are placed at fixed bearings from the sensor, each sampled at
three **noise levels** (`none` / `low` = 3 mm / `high` = 15 mm, Gaussian, along the true normal — 15 mm
deliberately matches project [`02.01`](../../02.01-voxel-grid-downsampling-with-gpu-spatial-hashing/data/README.md)
/ [`02.05`](../../02.05-kd-tree-or-lbvh-construction-knn-radius-search/data/README.md)'s own cited LiDAR
range-noise magnitude). That is **12 cohorts × 700 points = 8,400 points total**, each cohort spatially
well separated (tens of meters apart) so no point's K=16 neighborhood ever straddles two cohorts by
accident. See `../scripts/make_synthetic.py`'s module docstring for the full derivation of every surface's
geometry (placement, visible-region sampling, and why the edge cohort's perpendicular extent is
deliberately narrow) — summarized here:

- **Plane** (curvature ≡ 0 everywhere) — a 3 m × 3 m patch, deliberately tilted ~14° off the sensor's
  viewing axis so its normal is a general 3-vector, not an axis-aligned special case.
- **Sphere** (curvature ≡ 1/radius = 1.0 m⁻¹) — radius 1.0 m, sampled on the sensor-facing cap
  (half-angle 80°) with area-uniform density (no pole bunching).
- **Cylinder** (curvature ≡ 1/radius = 1.0 m⁻¹ in ONE principal direction; 0 along the axis) — radius
  1.0 m, sampled on the visible ±80° arc.
- **Edge** (**no single true curvature** — a genuine C⁰ discontinuity) — two half-planes meeting at a
  90° ridge; every point's true normal is its OWN face's normal, written with the sentinel curvature
  `-1.0`. This is the project's dedicated **degeneracy-detection** cohort: its perpendicular extent from
  the ridge (`EDGE_T_MAX_M = 0.05` m) is deliberately kept comparable to a K=16 neighborhood radius so a
  clear majority of its points genuinely straddle both faces.

Every point's world position, exact analytic normal, exact curvature (or the edge sentinel), and a
precomputed **grazing-angle cosine** (`|cos(true_normal, direction-to-sensor)|`, evaluated at the clean
pre-noise position) are written to the file — the grazing cosine is what `../src/main.cu`'s `GATE
orientation` and every angular-error gate use to separate "confidently viewable" points from the honest
grazing-incidence failure mode (THEORY.md derives why).

### Fields / format (`normals_scan.bin`)

A small hand-rolled binary format (no library — CLAUDE.md's no-black-boxes stance), read by
`../src/main.cu`'s `load_sample()` with **explicit fixed-width primitive reads** (never a raw struct
`fread`, so the layout never depends on a compiler's struct-padding rules), matching
[`02.01`](../../02.01-voxel-grid-downsampling-with-gpu-spatial-hashing/data/README.md)/[`02.05`](../../02.05-kd-tree-or-lbvh-construction-knn-radius-search/data/README.md)'s
identical discipline:

| Bytes | Type | Field | Meaning |
|-------|------|-------|---------|
| 0–7 | `char[8]` | magic | literal `NRMLCRV1`, no null terminator |
| 8–11 | `int32` | `n_points` | total point count (8,400) |
| 12–15 | `int32` | `n_cohorts` | cohort count (12) |
| 16–19 | `int32` | `k_neighbors` | the K this sample was designed against (16) — **must equal** `kernels.cuh`'s `kK`; `main.cu` asserts this at load time |
| 20–31 | `float32[3]` | `sensor_x, sensor_y, sensor_z` | the sensor origin, meters, sensor frame (0, 0, 0) |
| 32 … | 12 × 32-byte records | cohort table | one record per (surface, noise) cohort, in generation order — see below |
| … | `float32[n_points*3]` | `xyz` | interleaved x,y,z, **meters**, sensor frame (origin at the sensor, +x forward, +z up — CLAUDE.md §12), **possibly noisy** |
| … | `float32[n_points*3]` | `true_normal` | interleaved x,y,z, unit vector, the surface's EXACT analytic normal at the point's clean (pre-noise) position, already oriented toward the sensor |
| … | `float32[n_points]` | `true_curvature` | 1/meters; `0.0` on the plane; `1/radius` on the sphere/cylinder; **`-1.0` sentinel** on the edge cohort (no single true curvature exists at a discontinuity) |
| … | `float32[n_points]` | `grazing_cos` | `|cos(true_normal, direction-to-sensor)|`, unitless in `[0,1]`, evaluated at the clean position |

**Cohort table record** (32 bytes each, 12 records, generation order = surface-major
plane→sphere→cylinder→edge, noise-minor none→low→high within each surface — `main.cu`'s printed gate
output relies on this exact order):

| Offset (within record) | Type | Field | Meaning |
|-------|------|-------|---------|
| 0–3 | `int32` | `surface_id` | `0`=plane, `1`=sphere, `2`=cylinder, `3`=edge — must match `kernels.cuh`'s `kSurfacePlane`..`kSurfaceEdge` |
| 4–7 | `int32` | `noise_level` | `0`=none, `1`=low (3 mm), `2`=high (15 mm) |
| 8–11 | `int32` | `start` | first point index (inclusive) of this cohort, into the `xyz`/`true_normal`/… arrays |
| 12–15 | `int32` | `count` | number of points in this cohort (700) |
| 16–19 | `float32` | `param` | sphere/cylinder radius, meters (1.0); `0.0` for plane/edge |
| 20–31 | `float32[3]` | `axis_x, axis_y, axis_z` | the cylinder's true axis unit vector (GATE `cylinder_axis` compares a GPU-fitted axis against this); `(0,0,0)` for non-cylinder cohorts |

> **Synthetic, labeled everywhere.** Every point in this file is analytically generated (closed-form
> surfaces sampled from a simulated sensor, with an explicit, documented noise model) — nothing here is a
> recording of any real sensor or real space. `main.cu`'s `DATA:` line and every artifact this project
> writes say so explicitly.

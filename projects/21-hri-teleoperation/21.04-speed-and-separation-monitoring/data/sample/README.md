# `data/sample/` — the tiny committed sample

This folder holds the **small, committed, offline-runnable** sample the demo consumes — the reason
anyone can clone this repo, build, and see the documented result with **zero downloads** (CLAUDE.md
§4, §8). Rules of the folder:

- **Tiny.** Kilobytes preferred, megabytes tolerated, ~50 MB is the hard ceiling (and means you
  should rethink). The demo must run fast and the repo must stay clonable.
- **Labeled.** Synthetic data says so; public data names its source and license — both in
  [`../README.md`](../README.md), which documents provenance, checksums, and every field's meaning,
  units, and frame.
- **Reproducible.** Everything here can be regenerated by `../../scripts/make_synthetic.py` (fixed
  seed) or re-fetched by `../../scripts/download_data.ps1` — the committed bytes are a convenience,
  never the only copy of anything.
- **Self-contained.** If this project consumes data produced by a sensor-sim sibling project, the
  sample is **copied into** this folder — never referenced across project folders at build or run
  time (CLAUDE.md §4 self-containment rule).

**This project's sample:** `ssm_scenario.csv` (613 bytes, synthetic) — the task definition: how many
frames the sequence runs, at what rate, and where the human's walk starts and turns around. Every
depth pixel, robot pose, and human capsule position is synthesized in closed form at run time from
this tiny scenario plus the compile-time model in `../../src/kernels.cuh` (link lengths, camera
geometry, SSM formula parameters) — a controller-shaped project's data *is* its scenario, the same
split project 08.01 uses. Format and checksum: [`../README.md`](../README.md); regenerate
byte-identically with `python ../../scripts/make_synthetic.py`.

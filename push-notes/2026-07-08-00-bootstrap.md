## Summary

This is the bootstrap push (CLAUDE.md §11 Phase 0): it creates the entire skeleton of
**cuda-cpp-robotics-projects**, a didactic study collection of **505 self-contained CUDA C++
robotics projects across 36 domains**. Nothing here is a finished project yet — this push ships
the *contract and the scaffolding*: the repository-wide teaching standards, the shared robot
architecture every project will plug into, the tooling that parses the catalog and stamps out
project skeletons, the canonical project template, and one pre-filled skeleton per catalog entry.
For the learner, this push matters because it fixes, up front and in writing, *how* every future
project will teach: over-commented code, physics-first theory, a hardware/business companion, a
CPU-verified GPU demo, and an explicit place in a real robot.

## How to build & run

Nothing runnable ships in this push (skeletons only). To prepare the machine for what comes
next, follow [docs/BUILD_GUIDE.md](../docs/BUILD_GUIDE.md): Visual Studio 2026 (v145 toolset,
*Desktop development with C++*) + CUDA Toolkit 13.3; GPU floor is sm_75 (Turing). The repo
tooling runs today with Python 3.12:

```powershell
python tools/catalog.py          # regenerate catalog.json from the catalog markdown
python tools/status.py --check   # print the work-queue summary without writing
python tools/verify_project.py projects/<domain>/<SS.NN>-<slug>/   # gate a project
```

## Verification

All checks below were run on 2026-07-08 on the owner's machine — NVIDIA GeForce RTX 2080 SUPER
(sm_75, 8 GB), driver 591.86, CUDA Toolkit 13.3 (nvcc V13.3.33), Visual Studio 2026 Community
(v145 toolset), Windows 11:

- **Catalog:** `tools/catalog.py` parses the catalog to exactly **505 projects / 36 domains**
  with deterministic positional IDs; per-domain counts hand-checked against the markdown.
- **Template battle test (project 33.01):** scaffolded via `tools/scaffold.py --only 33.01`,
  then built through the generated `build/batched-small-matrix-linear-algebra.sln` with MSBuild
  in **both `Release|x64` and `Debug|x64` — zero errors, zero warnings** (after one template
  fix, see below). `demo/run_demo.ps1` ran the SAXPY placeholder on the GPU end-to-end:
  `RESULT: PASS (GPU matches CPU reference, max |gpu-cpu| <= tol 1e-6)`, all 3 expected lines
  matched `demo/expected_output.txt`, exit 0.
- **Full scaffold:** all **505 skeletons stamped** (504 created + the pre-tested 33.01 kept by
  the skip-if-exists rule); zero unsubstituted `{{TOKEN}}` placeholders repo-wide; edge cases
  spot-checked (26.12 dual-tag badge `★ beginner · [R&D]`, 30.01 bundled-bullet milestone note).
- **Gates:** `tools/verify_project.py` on the 33.01 skeleton passes every structural gate and
  fails exactly one — remaining `TODO(scaffold):` markers — which is the *designed* not-done
  state for a fresh skeleton.
- **Template fix ratified during testing:** CLAUDE.md §5 says Debug enables `-G` *and*
  `-lineinfo`, but nvcc/ptxas warn that `-G` overrides `-lineinfo` (device debug already embeds
  line info), which would violate the §9 zero-warnings gate. The template therefore sets only
  `-G` in Debug and `-lineinfo` in Release, with the reasoning commented in the `.vcxproj` —
  surfacing the §5 wording conflict here rather than silently working around it (§13).
